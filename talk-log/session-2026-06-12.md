# Session Log — 2026-06-12 / 2026-06-13

## Context

Reviving the Defcoin (DFC) network, originally created for DEF CON 22 in 2014 as "the official hacker crypto currency." Single Azure VM (Standard_B2s, 40.87.31.48) running defcoind v26.3.1, UNOMP mining pool at pool.defcoin.fun:3333, and eIquidus explorer at explorer.defcoin.fun.

---

## What we fixed

### blocknotify.sh — pool master crashing on new blocks
Script was sending plain text to UNOMP's CLI listener. CLI called JSON.parse() on it, threw uncaught exception, crashed the master process.

**Fix:** Rewrote script to send correct JSON format:
```bash
printf '{"command":"blocknotify","params":["defcoin","%s"],"options":{}}\n' "$1" | nc -q 1 127.0.0.1 17323
```

Also moved the try-catch in cliListener.js to *inside* the data callback (it was wrapping the event registration, not the handler — useless there).

### Explorer 502 bad gateway
Workers crashing on every request: `ReferenceError: crypto is not defined`

**Fix:** Node.js 16 needs `--experimental-global-webcrypto` flag. Was set for the sync service but not the web service. Added to defcoin-explorer.service:
```
Environment="NODE_OPTIONS=--experimental-global-webcrypto"
```

### Network hashrate showing 0
`nethash_units` was set to "T" (terahash). Defcoin network does ~3 MH/s. 3 MH/s / 1 TH = ~0.

**Fix:** Changed to "M" (megahash).

### Node not advertising itself to peers
`getnetworkinfo` showed `localaddresses: []` — node didn't know its external IP.

**Fix:** Added `externalip=40.87.31.48` to defcoin.conf.

### P2P port mismatch
Defcoin chainparams defaults to 17333, but the active network uses 1337. Our node was unreachable to most peers.

**Fix:** Added `port=1337` to defcoin.conf. Immediately got first-ever inbound connection from 70.124.241.251 (Lion-alpha node). Connections went from 3 to 6.

## Block propagation analysis

Analyzed 1,650 blocks from defcoind debug.log:

| Metric | Before port 1337 | After port 1337 |
|---|---|---|
| Median delay | 61s | 42s |
| Mean delay | 157s | 93s |
| Blocks >2min late (guaranteed orphan) | 29% | 23% |
| Max delay | 3599s (1 hour batch dump) | 1021s |

The 1-hour max was a peer reconnection event on 2026-06-10 where a peer dumped ~8,000 blocks in 30 minutes after reconnecting.

**Key finding:** 23% of blocks arrive more than 2 minutes after they were produced. Since defcoin has 2-minute blocks, these windows guarantee any block we submit during them is orphaned. We found 20 blocks total; 18 were orphaned.

## Things that don't work / gotchas

- **`$C` in the RPC password** expands to empty string in bash heredocs. Always write defcoin.conf via Python.
- **defcoin.conf changes require defcoind restart** — addnode RPC changes are runtime-only.
- **Writing JS files with `\n` via SSH heredoc** is unreliable. Use base64: encode on Windows, pipe through `base64 -d` on VM.
- **eIquidus network page** requires both `network_page` section in settings.json AND running `node scripts/sync.js peers reindex` to populate MongoDB.
- **getinfo RPC was removed in Bitcoin Core v26** — UNOMP calls it. Patched pool.js and daemon.js to synthesize from getblockchaininfo.
- **bignum npm package** won't compile on Node 12+. Replaced with pure-JS BigInt, must honor `{endian: 'little'}` option or every share is rejected as low difficulty.

## The orphan problem (unsolved)

Root cause: with ~5 active nodes on the whole network, block propagation is slow. Even with blocknotify and port 1337, median delay is 42 seconds on a 2-minute block chain. 

More relay nodes and better peer connectivity help marginally. The real fix would be more hashrate (>50% of ~3-5 MH/s network) or direct peering with all active miners. Given defcoin was built as a toy for DEF CON 22, the "competition" is a handful of nodes running scrypt CPUs.
